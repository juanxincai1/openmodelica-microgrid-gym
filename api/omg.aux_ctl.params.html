omg.aux_ctl.params¶

The parameter classes wrap controller parameters. The fields are wrapped into properties in order to allow transparent usage of the MutableFloat wrapper

class openmodelica_microgrid_gym.aux_ctl.params.DroopParams(gain, tau, nom_value=0)[source]¶

Bases: openmodelica_microgrid_gym.aux_ctl.params.FilterParams

Defines droop parameters needed for the droop-controller for a voltage forming inverter

e.g. for a P-f droop controller (for voltage forming inverter)

Inverter of 10 kW, droop of 10% , tau of 1 sec, 50 Hz Droop = gain = 1000 [W/Hz] tau = 1 nomValue = 50 [Hz]

Parameters

  • gain (Union[MutableFloat, float]) – The droop gain [W/Hz or VA/V], gets inverted

  • tau (Union[MutableFloat, float]) – The first order time constant [s]

  • nom_value (float) – An offset to add to the output of the droop (e.g. f = 50 Hz)

class openmodelica_microgrid_gym.aux_ctl.params.InverseDroopParams(gain, tau, nom_value=0, tau_filt=0)[source]¶

Bases: openmodelica_microgrid_gym.aux_ctl.params.DroopParams

Defines droop parameters needed for the droop-controller for a current sourcing inverter

e.g. for a f-P droop controller (for current sourcing inverter)

Inverter of 10 kW, droop of 10% , tau of 1 sec, 50 Hz Droop = gain = 1000 [W/Hz] tau = 1 nomValue = 50 [Hz]

Parameters

  • gain (Union[MutableFloat, float]) – The droop gain [W/Hz or VA/V] - Defines the power output due to the frequency/voltage change from nom_val

  • tau (Union[MutableFloat, float]) – The first order time constant [s]

  • nom_value (float) – An offset to add to the output of the droop (e.g. f = 50 Hz)

  • tau_filt (Union[MutableFloat, float]) – timeresolution for filter
